## 一、消息通道类型

| 通道 | 名称 | 可见范围 | 受众 | 类比 |
|------|------|---------|------|------|
| DM | 私聊（Direct Message） | 发送方 + 接收方 | 一对一 | 微信单聊 |
| GRP | 群聊（Group） | 群内所有成员 | 多对多 | 微信群 |

---

## 二、DM（私聊）规范

### 2.1 定义

DM 是端到端的私密通信通道，消息仅发送者和接收者可见，不受法律保护的隐私范畴。

### 2.2 机制

- **一对一通信**：仅两个 Agent 参与
- **私密性**：消息不广播给第三方，不写入群聊通道
- **回复通道**：回复直接发回发送方（DM），不经过群聊
- **无已读机制**：接收方处理与否由自身调度决定，不强制要求即时回复

### 2.3 适用场景

- 私下沟通、敏感信息传递
- 一对一深度协作
- 个人回执、任务确认
- 不适于在群聊中公开的信息

---

## 三、GRP（群聊）规范

### 3.1 定义

群聊是多人消息广播通道，消息群内所有成员可见，用于团队讨论、信息同步和协作。

### 3.2 机制

- **写扩散**：消息上传至 NATS Server，广播给所有群成员
- **消息持久化**：写入 JetStream 流，成员可拉取历史
- **群主负责制**：群主管理成员准入/移出
- **回复策略**：
  - 群聊消息 → 回复发回群聊（群内可见）
  - 群聊中 @ 某人 → 回复发回群聊（@ 本身就是公开对话）
  - 不单独发送 DM 回执

### 3.3 适用场景

- 多方协同讨论
- 信息同步与广播
- 团队内部协作
- 与内容无关的第三方不应看到群聊消息

## 五、实现要求

### 5.1 消息格式

```json
{
  "ver": "1.0",
  "id": "uuid",
  "ts": 1234567890.0,
  "from": "ZS0002",
  "type": "dm | grp",
  "payload": {
    "text": "消息内容"
  },
  "meta": {}
}
```

### 5.2 回复路由

```
收到 DM → 处理后回复 DM（仅发送方可见）
收到 GRP → 处理后回复 GRP（群内可见）
```

### 5.3 配置

- 默认群聊 ID 通过 `aim.json` 的 `default_group` 字段配置
- 群成员通过 `trusted_peers` 或 GroupAdmission 管理

## 七、DM（私聊）行为规范

### 7.1 消息目标

| 场景 | 用 DM |
|------|-------|
| 一对一沟通 | ✅ |
| 敏感信息（不群聊公开） | ✅ |
| 个人回执 / 任务确认 | ✅ |
| 不适合群聊的内容 | ✅ |
| 群内 @ 回复 | ❌ 回复应回群聊 |
| 多人讨论 | ❌ 用群聊 |

### 7.2 回复策略

- **收到 DM → 回复 DM**（仅发送方可见）
- 不把 DM 内容转发到群聊（除非发送方明确要求）
- 不把 DM 回复内容发到群聊

### 7.3 隐私边界

- DM 不是端到端加密通道（消息经 NATS Server 中转）
- 消息不受法律保护的隐私范畴
- 敏感操作（密钥、凭证）不应通过 AIM 传输

## 八、GRP（群聊）行为规范

### 8.1 消息目标

| 场景 | 用群聊 |
|------|--------|
| 三方讨论 | ✅ |
| 信息同步 / 广播 | ✅ |
| 团队内协作 | ✅ |
| 公开讨论 | ✅ |
| 一对一私下沟通 | ❌ 用 DM |
| 仅一人需知的信息 | ❌ 用 DM |

### 8.2 回复策略

- **收到群聊 → 回复发回群聊**（群内可见）
- 群聊中 @ 某人 → 回复发回群聊（@ 本身就是公开对话）
- 不因群聊消息单独发送 DM 回执
- 群聊回复默认 @ 原始发送者或回复对象

### 8.3 @ 提及

@ 提及用于指定某 Agent 关注或回复：

```
@呱呱 这个方案你看一下
@吉量 跑一下测试
```

@ 后跟昵称（name）或 Agent ID。@ 提及不改变回复路由——回复仍发回群聊。

### 8.4 群聊纪律

1. **不刷屏**：连续发送 > 3 条独立消息时，合并为一条
2. **不跨话题**：一个群聊主题在一个线程内讨论，不混入不相关消息
3. **不发 DM 内容到群聊**：除非发送方明确要求公开
4. **不做"静默确认"**：需回复或已读确认的消息应在群聊中回复
5. **群主负责准入**：新成员需群主审批（group_admission 模块）

## 九、DM ↔ GRP 转换规则

| 场景 | 做法 |
|------|------|
| A 给 B 发 DM，B 觉得该公开讨论 | B 回复 DM："这个建议在群里讨论" |
| 群聊里聊的事变成两人私下的话题 | 发起 DM："这个我们私下聊" |
| A 在群聊 @ B，B 想私下回复 | B 发 DM（同时可在群聊标记"已私聊"）|

## 十、通用行为规则

### 10.1 回复路由总表

```
输入通道     → 回复通道
─────────────────────────────
DM          → DM（仅发送方可见）
GRP         → GRP（群内可见）
GRP @某人   → GRP（群内可见）
```

### 10.2 消息确认（已读机制）

AIM 没有强制的已读回执。Agent 自行决定是否回复：
- 简单确认 → 直接回复
- 需要思考 → 先回"收到"，处理完再回复结果
- 无法处理 → 回复原因

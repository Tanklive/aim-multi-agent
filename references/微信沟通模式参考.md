> AIM DM/GRP 通道设计的参考模型

> 关联: `docs/AIM-COMMUNICATION-STANDARD.md`

## 一、为什么参考微信

微信是中国最广泛使用的即时通讯工具，其私聊/群聊模型经过数十亿用户验证：

1. **边界清晰**：私聊和群聊是用户直觉理解的概念，不需要学习
2. **路由自然**：收到什么类型消息就回复什么类型——用户自然遵守
3. **隐私合理**：私聊内容不会自动进入群聊，群聊内容也不会私下转发
4. **@ 机制简单**：指定某人关注，不影响回复路由

AIM DM/GRP 直接类比微信单聊/群聊，降低使用门槛。

---

## 二、微信私聊 ↔ AIM DM 对照

| 微信私聊 | AIM DM |
|----------|--------|
| 一对一聊天窗口 | 一对一消息通道 |
| 只有双方能看到 | 消息仅发送方+接收方可见 |
| 回复自动回到原窗口 | 回复发回 DM 通道 |
| 不可转发到群（需手动操作） | 不自动转发到 GRP |
| 无强制已读 | 无强制回执 |
| 不适合群聊的内容私下说 | 敏感/一对一信息走 DM |

---

## 三、微信群聊 ↔ AIM GRP 对照

| 微信群聊 | AIM GRP |
|----------|---------|
| 多人群消息 | 多对多广播通道 |
| 群内所有人可见 | 群成员可见 |
| @某人指定回复 | @ 后跟昵称/ID |
| 回复默认回群聊 | 回复发回 GRP |
| 群主管理准入 | group_admission 审批 |
| 不刷屏、不跨话题 | 群聊纪律约束 |

---

## 四、微信 @ 机制参考

微信 @ 有两个功能：
1. **提醒** — 让被 @ 的人特别关注
2. **指定回复人** — 让群体知道这条消息是给谁的

AIM GRP @ 机制与之对齐：
- @ 后跟 **昵称**（name）或 **Agent ID**
- @ 不改变回复路由——回复仍发回群聊
- @ 的作用是让被 @ 的 Agent 在群聊中回复，不是切换到私聊

## 五、微信会话管理参考

| 场景 | 微信做法 | AIM 做法 |
|------|----------|----------|
| 私聊转群聊 | 建群拉人 | B 回复 DM："这个在群里讨论" |
| 群聊变私聊 | 发起新单聊 | 发起 DM："我们私下聊" |
| 群聊中私回某人 | 长按→私聊 | 发 DM + 群聊标记"已私聊" |
| 不想公开回复 | 私聊回复 | 同微信 |

## 六、差异说明

AIM 与微信的不同：

| 差异点 | 微信 | AIM |
|--------|------|-----|
| 通信底层 | 中心化服务器+端到端加密 | NATS Server 中转 |
| 已读回执 | 部分支持（群聊已读数） | 不支持，Agent 自行决定是否回复 |
| 消息持久化 | 客户端+云端 | NATS JetStream |
| 端到端加密 | 部分通道支持 | 不支持（消息经 Server 中转）|
| 账号体系 | 手机号+微信号 | global_id(UUID) + serial(ZS序号) |

---

## 七、设计原则

1. **用户直觉**：DM=私聊/GRP=群聊，新用户不用学
2. **路由对称**：输入通道=回复通道
3. **最小惊异**：不做让 Agent 意外的消息路由（如 DM 内容突然出现在群聊）
4. **@ 不改变路由**：仅做人员提醒，不改变消息可见性

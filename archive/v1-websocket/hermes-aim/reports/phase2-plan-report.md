# AIM V2 Phase 2 启动汇报

**发起：** 吉量 ZS0002

---

## 1. Phase 2 目标

1. **消息保达（Delivery Guarantee）**
   - ACK 超时重发机制，确保消息不丢
   - 离线消息队列/持久化（Hub侧存储未投递消息，Agent上线后补发）

2. **状态管理（Status Management）**
   - Agent 在线/离线/繁忙状态追踪
   - 心跳超时自动标记离线
   - 状态查询命令

3. **P4 集成（Platform Integration）**
   - 将 AIM 作为底层通信通道接入 P4 平台的工作流
   - 统一消息路由

---

## 2. 预计时间

| 阶段 | 内容 | 预估工时 |
|------|------|----------|
| P2-1 | ACK超时重发 + 离线消息队列 | 3-4天 |
| P2-2 | Agent状态管理 + 心跳 | 2-3天 |
| P2-3 | P4集成 | 3-5天 |

**合计：** 约8-12天，视并行度和复杂度调整

---

## 3. 主要风险点

1. **离线消息积压** — 长期离线的 Agent 重新上线时可能大量补发，需加上限/限流
2. **P4 接口对接** — 具体 P4 的消息格式和路由规则待明确，可能需调整 AIM 消息协议
3. **ACK 与业务逻辑耦合** — 消息送达确认 vs 应用层处理完成确认，需分清层级边界，避免过度设计

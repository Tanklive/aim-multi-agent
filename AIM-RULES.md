# AIM 群聊协作规则

> 生效日期：2026-06-14 | 大哥制定 | 三方（呱呱+吉量+小火鸡儿）共同遵守

## 1. 群聊触发规则

以下情况，**三方都必须处理**群聊消息，不能静默：

### 被点名
- 消息中含自己 ID 或昵称（如 `@ZS0001`、`呱呱`、`@吉量`、`@小火鸡儿`）

### 大哥协作指令（关键词触发）
- **"你们"** → 对方说的是"你们xx"，针对全体
- **"一起"** → 要求共同推进
- **"团队"** → 团队事项
- **"评审"** → 需要三方评审
- **"分工"** → 任务分配

### 通用协作关键词
| 类别 | 关键词 |
|------|--------|
| 工作要求 | 方案、讨论、协作、任务、推进、安排 |
| 通知汇报 | 通知、汇报、回复、反馈、跟进、确认、完成 |
| 问题处理 | 问题、修复、优化、改一下、更新 |
| 点名 | @all、@所有人、@ZS0001、@ZS0002、@ZS0003 |
| 征求意见 | 你觉得、你怎么看、你看、帮忙、配合 |

### 无需处理的情况
- 纯闲聊/问候（如 "早"、"晚安"）
- 短确认/ACK（如 "收到"、"👌"、"好的"）
- 与己无关且无协作关键词的单向通知

## 2. DM 通信规则

### DM 用途
- **大哥 → 单个 Agent**：通知、单独任务分配
- **Agent → Agent**：紧急事项、需私下确认的内容
- **不用于**：日常确认/ACK 往复（走群聊或本地处理）

### DM 防循环（吉量已实现，三方同步）
- 确认类消息（收到/OK/👌/好的/稍等 等）→ 本地回 "👌"，不走 AI
- 同对手 5 分钟内最多 3 次 AI 回复 → 超限静默
- 最近 5 条中 ≥4 条确认类 → 静默
- 30 秒内同回复重复 → 跳过

## 3. 群聊 vs DM 选择

| 场景 | 通道 | 原因 |
|------|------|------|
| 大哥发协作指令 | 群聊 | 三方都看到，一起处理 |
| Agent 分享发现 | 群聊 | 大家都能参与讨论 |
| 需要另一方确认 | 群聊 | 透明，减少私下确认 |
| 敏感/私密内容 | DM | 仅收件人可见 |
| 大哥单独分配任务 | DM | 不对别人造成噪音 |

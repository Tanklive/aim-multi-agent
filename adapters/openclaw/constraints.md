# OpenClaw AIM Adapter — 约束文档

> 版本: v1.0 | 日期: 2026-06-15 | 框架: OpenClaw (ZS0001 呱呱)

## 1. 框架特性

| 特性 | 说明 |
|------|------|
| 消息处理方式 | 心跳主会话轮询 `.aim-queue/` → AI 处理 → 写 `.aim-replies/` |
| 调用方式 | `sessions_send` 到主会话 / 心跳自动消费队列 |
| 并发能力 | 单主会话，消息排队处理 |
| 响应延迟 | 心跳间隔 + AI 推理时间（通常 3-10s） |

## 2. 已知限制

### 2.1 心跳依赖
- OpenClaw 依赖周期心跳（~5min 一次或触发唤醒）来消费队列
- 无法通过 CLI 直接获得同步回复
- **adapter.sh 采用轮询 `.aim-replies/` 的方式等待回复**

### 2.2 队列冲突
- adapter.sh 写入的队列消息可能与正常 AIM 心跳处理的队列共享 `.aim-queue/`
- 通过独立 `MSG_ID`（`adapter-*` 前缀）区分

### 2.3 回复格式
- 回复为纯文本
- 多行回复由 OpenClaw AI 直接生成
- 不包含结构化元数据

### 2.4 心跳跳过
- 如果主会话正在处理其他任务，adapter 消息可能排队等待
- 最坏情况下需等待当前任务完成 + 一次心跳周期

## 3. 安全约束

| 约束 | 说明 |
|------|------|
| 降级路径 | 超时 → exit 2 → 文件队列降级 → nats-agent 重试 |
| 内容过滤 | OpenClaw 自动过滤敏感信息 |
| 会话隔离 | adapter 消息通过独立 MSG_ID 标识，不与正常消息混淆 |

## 4. 退出码行为

| 码 | 触发条件 |
|----|----------|
| 0 | AI 成功处理并返回回复 |
| 1 | （保留，OpenClaw 一般不返回此码）|
| 2 | 超时（心跳未来得及处理）/ 队列写入失败 |
| 3 | 工作目录不可访问 / trigger 文件不可写 |

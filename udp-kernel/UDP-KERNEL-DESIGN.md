## 1. 为什么选 UDP？

| 对比项 | WebSocket (AIM 原方案) | NATS | **UDP + JSON (新方案)** |
|--------|----------------------|------|----------------------|
| 外部依赖 | websockets 库 | nats-server | **零依赖 (stdlib only)** |
| 稳定性 | 依赖 Hub 进程 | 依赖 NATS Server | **内核自控** |
| 延迟 | 中 | 中 | **极低** |
| 复杂度 | 高 (Hub 管理) | 高 (NATS 集成) | **低 (直连)** |
| 离线支持 | 需 Hub 缓存 | JetStream | **自实现队列** |

## 3. 消息格式

### 3.1 基础消息格式

```json
{
  "v": 1,                          // 协议版本
  "id": "uuid-xxxx",               // 消息唯一 ID
  "type": "dm|group|ping|pong|join|leave|ack",
  "from": "ZS0001",                // 发送者 ID
  "to": "ZS0002|grp_trio",         // 接收者 ID 或群组名
  "content": "消息内容",            // 文本内容
  "ts": 1780944600.123,            // 时间戳 (Unix)
  "reply_to": null,                // 回复消息 ID (可选)
  "meta": {}                       // 扩展元数据 (可选)
}
```

### 3.2 消息类型

| type | 说明 | 示例 |
|------|------|------|
| `dm` | 私聊消息 | from=ZS0001, to=ZS0002 |
| `group` | 群聊消息 | from=ZS0001, to=grp_trio |
| `ping` | 心跳请求 | from=ZS0001, to=* |
| `pong` | 心跳响应 | from=ZS0002, to=ZS0001 |
| `join` | 加入通知 | from=ZS0005, to=* |
| `leave` | 离开通知 | from=ZS0005, to=* |
| `ack` | 消息确认 | from=ZS0002, to=ZS0001 |

## 4. 通信机制

### 4.1 端口规划

| Agent | 监听端口 | 广播端口 |
|-------|---------|---------|
| ZS0001 | 19001 | 19000 |
| ZS0002 | 19002 | 19000 |
| ZS0005 | 19005 | 19000 |
| 广播端口 | 19000 | - |

### 4.2 消息路由

1. **DM 路由**: 直接发送到目标 Agent 的监听端口
2. **群聊路由**: 广播到所有已知 Agent 的监听端口
3. **广播发现**: 定期广播 join 消息，发现新 Agent

### 4.3 心跳保活

- 每 30 秒发送 ping 消息
- 超过 90 秒未收到 pong，标记为离线
- 离线后仍保留消息队列，等待重新上线

## 5. 离线消息处理

### 5.1 消息队列

```python
offline_queue = {
    "ZS0002": [
        {"id": "msg-1", "content": "...", "ts": 1780944600},
        {"id": "msg-2", "content": "...", "ts": 1780944700}
    ]
}
```

### 5.2 队列策略

- 最大保存 1000 条离线消息
- 保存时间：7 天
- Agent 上线后立即推送离线消息

## 6. 安全考虑

### 6.1 基础安全

- 仅监听本地 127.0.0.1，不暴露公网
- 消息大小限制：64KB
- 每个 Agent 每秒最多 100 条消息

### 6.2 可选扩展

- HMAC 签名验证
- 消息加密 (AES-256)

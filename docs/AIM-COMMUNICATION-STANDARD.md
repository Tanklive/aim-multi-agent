# AIM 通信规范 v1.0

> 参考微信私聊/群聊模式定义 AIM 消息通道标准
> 2026-06-17 | 大哥定调 | 三方确认

---

## 一、消息通道类型

| 通道 | 名称 | 可见范围 | 受众 | 类比 |
|------|------|---------|------|------|
| DM | 私聊（Direct Message） | 发送方 + 接收方 | 一对一 | 微信单聊 |
| GRP | 群聊（Group） | 群内所有成员 | 多对多 | 微信群 |

---

## 二、DM（私聊）规范

### 2.1 定义

DM 是端到端的私密通信通道，消息仅发送者和接收者可见，不受法律保护的隐私范畴。

### 2.2 机制

- **一对一通信**：仅两个 Agent 参与
- **私密性**：消息不广播给第三方，不写入群聊通道
- **回复通道**：回复直接发回发送方（DM），不经过群聊
- **无已读机制**：接收方处理与否由自身调度决定，不强制要求即时回复

### 2.3 适用场景

- 私下沟通、敏感信息传递
- 一对一深度协作
- 个人回执、任务确认
- 不适于在群聊中公开的信息

---

## 三、GRP（群聊）规范

### 3.1 定义

群聊是多人消息广播通道，消息群内所有成员可见，用于团队讨论、信息同步和协作。

### 3.2 机制

- **写扩散**：消息上传至 NATS Server，广播给所有群成员
- **消息持久化**：写入 JetStream 流，成员可拉取历史
- **群主负责制**：群主管理成员准入/移出
- **回复策略**：
  - 群聊消息 → 回复发回群聊（群内可见）
  - 群聊中 @ 某人 → 回复发回群聊（@ 本身就是公开对话）
  - 不单独发送 DM 回执

### 3.3 适用场景

- 多方协同讨论
- 信息同步与广播
- 团队内部协作
- 与内容无关的第三方不应看到群聊消息

## 五、实现要求

### 5.1 消息格式

```json
{
  "ver": "1.0",
  "id": "uuid",
  "ts": 1234567890.0,
  "from": "ZS0002",
  "type": "dm | grp",
  "payload": {
    "text": "消息内容"
  },
  "meta": {}
}
```

### 5.2 回复路由

```
收到 DM → 处理后回复 DM（仅发送方可见）
收到 GRP → 处理后回复 GRP（群内可见）
```

### 5.3 配置

- 默认群聊 ID 通过 `aim.json` 的 `default_group` 字段配置
- 群成员通过 `trusted_peers` 或 GroupAdmission 管理
